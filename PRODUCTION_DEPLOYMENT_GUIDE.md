## 🐛 **既知の問題と回避策**

- **問題**: `grp --version` がエラーになる
- **回避策**: `grp --help` でバージョン情報確認
- **修正予定**: v2.0.1で修正

#### 2. **スマートフィルタリングのログ出力**
- **問題**: デバッグログでフィルタリング詳細が見えない
- **影響**: 機能は正常動作、ログのみの問題
- **修正予定**: v2.0.1で改善

### **パフォーマンス特性**
- **平均処理時間**: 4.4秒（50コメント程度のPR）
- **メモリ使用量**: 軽量（大きなPRでも安定）
- **API効率**: GitHub API制限内で安全動作

## 🔄 **トラブルシューティング**

### **よくある問題**

#### **1. GitHub認証エラー**
```bash
# トークンの確認
echo $GITHUB_TOKEN

# トークンの権限確認（以下が必要）
# - repo (プライベートリポジトリの場合)
# - public_repo (パブリックリポジトリの場合)
```

#### **2. 処理が遅い**
```bash
# デバッグモードで詳細確認
grp --debug [PR_URL] 2>&1 | grep -E "(INFO|WARNING|ERROR)"

# 大量コメントの場合は正常（10秒以内なら問題なし）
```

#### **3. 返信判定が不正確**
- **対処**: フィードバックをGitHub Issuesに報告
- **一時回避**: 手動で判定結果を確認・修正
- **改善**: ユーザーフィードバックで継続改善

### **ログレベル調整**
```bash
# 詳細ログ
grp --debug [PR_URL]

# 静寂モード
grp [PR_URL] 2>/dev/null
```

## 📈 **パフォーマンス最適化**

### **大量コメント処理**
- **100件以上**: 自動的にバッチ処理
- **500件以上**: 段階的処理（進捗表示）
- **1000件以上**: メモリ最適化モード

### **API制限対策**
- **レート制限**: 自動的な遅延・リトライ
- **バッチ処理**: 複数コメントを1回のAPIで処理
- **キャッシュ**: 重複データの削減
